# Плагин astg-hook

## Назначение

Плагин **astg-hook** сохраняет текущую модель проекта (описание контрактов) в **локальную базу контрактов**. После этого вы можете запускать пайплайн по сохранённой версии с помощью плагина **astg-db** и опции `from-db` — без повторного клонирования репозитория и разбора кода.

Плагин срабатывает автоматически после того, как в пайплайне уже получена модель проекта (например, плагином astg из исходников). Он просто дополняет пайплайн: сохраняет результат в базу для последующего использования.

## Когда это полезно

- Вы один раз разобрали репозиторий (или локальный проект), и хотите в дальнейшем генерировать код по этой же версии контрактов, не обращаясь снова к исходникам.
- Вы хотите накапливать несколько версий одного проекта (например, по разным веткам или тегам) и переключаться между ними через astg-db.
- Удобно комбинировать с astg-db: один запуск с разбором репо + astg-hook сохраняет в базу; следующие запуски с astg-db и `from-db` берут контракты из базы.

## Как работает с точки зрения пользователя

1. В пайплайн добавлен плагин **astg** (и при необходимости другие плагины, которые строят модель проекта).
2. В тот же пайплайн добавлен плагин **astg-hook** (он выполняется после astg).
3. Когда пайплайн получает модель проекта (например, astg разобрал контракты из репозитория или из кэша), astg-hook сохраняет эту модель в локальную базу. Идентификатор в базе определяется по проекту и версии (например, ветка или тег из Git, либо значение по умолчанию для локального проекта).
4. Если в этом же запуске вы загрузили проект **из базы** (передали опцию `from-db` в пайплайн, чтобы подставить контракты через astg-db), astg-hook **не сохраняет** модель в базу — чтобы не перезаписывать уже сохранённую версию.

Итого: при обычном запуске (разбор репо или локальных контрактов) модель сохраняется в базу; при запуске с загрузкой из базы (`from-db`) сохранение не выполняется.

## Опции

Плагин не добавляет своих опций для пользователя. Поведение зависит только от наличия в запросе опции **from-db**: если она передана (например, вы используете astg-db с `from-db`), astg-hook не сохраняет проект в базу.

## Зависимости

В пайплайне должен быть подключён плагин **astg**. astg-hook выполняется после него и сохраняет уже полученную модель.

## Связь с astg-db

Локальная база контрактов, которую заполняет **astg-hook**, используется плагином **astg-db**: с опцией `from-db` вы загружаете из этой базы нужный проект и версию (или выбираете из списка). Подробнее — в документации плагина astg-db.
