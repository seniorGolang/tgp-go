# Плагин astg-db

## Назначение

Плагин **astg-db** позволяет брать описание контрактов (модель проекта) из **локальной базы контрактов**, а не из исходников в репозитории. Это удобно, когда вы уже один раз сохранили проект в базу (с помощью плагина astg-hook) и хотите запускать пайплайн по сохранённой версии — без повторного клонирования или разбора кода.

Плагин срабатывает только если в запросе ещё **нет** готовой модели проекта и указана опция загрузки из базы.

## Когда использовать

- Нужно генерировать код (сервер, клиенты, Swagger и т.д.) по ранее сохранённой версии контрактов.
- Хотите выбрать проект или версию из списка в интерактивном режиме.
- Нужно взять только часть контрактов из сохранённого проекта (один или несколько по имени).

## Опция пайплайна

| Опция     | Тип    | Описание |
|-----------|--------|----------|
| `from-db` | строка | Загрузить проект из локальной базы контрактов. Значение — ссылка на сохранённый проект (см. ниже) или **пустая строка** для интерактивного выбора из списка. |

Если опция `from-db` не передана или в запросе уже есть модель проекта, плагин ничего не делает.

## Формат ссылки (ref)

Ссылка задаёт **проект**, **версию** и при необходимости **только нужные контракты**.

- **Проект и версия**  
  `имя_проекта@версия`  
  Примеры: `myapi@v1.0.1`, `myapi@main`, `myapi@default`.  
  Если версию не указать (`имя_проекта`), будет использована последняя сохранённая версия по дате обновления.

- **Проект, версия и контракты**  
  Один или несколько контрактов (по имени или ID) через запятую:
  - `имя_проекта:Контракт1,Контракт2@версия`  
  - или `имя_проекта@версия:Контракт1,Контракт2`  

  Пример: `myapi:UserService,OrderService@main` — из проекта `myapi` версии `main` загружаются только контракты `UserService` и `OrderService` (и связанные с ними сервисы). Версию можно опустить — тогда берётся последняя по дате.

## Интерактивный выбор

Если передать опцию `from-db` с **пустым** значением, плагин выведет список доступных в базе ссылок вида `проект@версия` и предложит выбрать одну. Если в базе ничего нет, плагин просто завершит работу без ошибки (пайплайн продолжит без подстановки проекта из базы).

## Откуда в базе берутся контракты

Локальная база контрактов заполняется плагином **astg-hook**: при запуске пайплайна с уже имеющейся моделью проекта (например, полученной из репозитория) astg-hook сохраняет её в базу. После этого вы можете использовать astg-db с опцией `from-db`, чтобы подставлять эту или другую сохранённую версию в последующие запуски.

## Зависимости

Для работы плагина astg-db в пайплайне должен быть подключён плагин **astg**.
