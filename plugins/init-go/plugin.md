# Плагин init-go — генератор базового Go-проекта

## Назначение

Плагин создаёт заготовку Go-проекта: структуру каталогов, контракты интерфейсов (JSON-RPC и/или REST), реализацию сервиса с заглушками методов и точку входа приложения. Транспорт (HTTP-сервер) и OpenAPI-описание генерируются отдельно после выполнения `go generate ./...` в созданном проекте.

## Команда

```bash
tg init go --module <имя_модуля> [--out <директория>] [--json-rpc <имена>] [--rest <имена>]
```

## Параметры

| Параметр    | Краткая форма | Обязательный | Описание |
|-------------|---------------|--------------|----------|
| `--module`  | `-m`          | да           | Имя Go-модуля (например, `myapp`). |
| `--out`     | `-o`          | нет          | Каталог, в который записать проект (относительно текущей рабочей директории). По умолчанию — текущий каталог. |
| `--json-rpc`| —             | нет          | Список имён JSON-RPC-интерфейсов через запятую (например, `some,demo`). |
| `--rest`    | —             | нет          | Список имён REST-интерфейсов через запятую (например, `example,siteNova`). |

Если не указать ни `--json-rpc`, ни `--rest`, будет создан один JSON-RPC-интерфейс с именем `Some`.

Имена интерфейсов приводятся к виду: тип в Go (PascalCase) и имя файла (snake_case). Например, `siteNova` даёт тип `SiteNova` и файл `site_nova.go` в каталоге `contracts/`.

## Ограничения

- Записываются только файлы внутри выбранного каталога вывода (`--out`). Корнем считается текущая рабочая директория при запуске команды.
- Каталог вывода должен **не существовать** либо **не содержать .go-файлов**. Иначе команда завершится ошибкой.

## Примеры

Создать проект с модулем `demo`, двумя JSON-RPC-интерфейсами (`Some`, `Demo`) и двумя REST-интерфейсами (`Example`, `SiteNova`) в текущем каталоге:

```bash
tg init go --module demo --json-rpc some,demo --rest example,siteNova
```

В `contracts/` появятся файлы `some.go`, `demo.go`, `example.go`, `site_nova.go` с объявлениями интерфейсов.

Только один JSON-RPC-интерфейс `Some` с заглушкой метода `Do`:

```bash
tg init go -m myapp --json-rpc some
```

Только один REST-интерфейс `Example` с заглушками CRUD (Create, Get, Update, Delete, List):

```bash
tg init go -m myapp --rest example
```

Создать проект в подкаталоге `my-service`:

```bash
tg init go -m some -o my-service --json-rpc some --rest example
```

## Структура созданного проекта

После успешного выполнения команды в каталоге вывода появляются:

- **go.mod** — объявление Go-модуля и перечень зависимостей (HTTP, логирование, конфигурация и др.).
- **contracts/** — контракты API:
  - **tg.go** — метаданные API и инструкции для последующей генерации транспорта и Swagger;
  - по одному **.go**-файлу на каждый указанный интерфейс (имя файла в snake_case: `some.go`, `site_nova.go` и т.п.);
  - **dto/** — при указании `--rest`: общие типы запросов и ответов для REST (CRUD).
- **internal/config/** — конфигурация сервиса (чтение из переменных окружения).
- **internal/services/** — пакеты с реализацией сервисов и заглушками методов (возвращают «не реализовано»).
- **internal/transport/** — заготовка транспорта; полный HTTP-сервер и маршруты создаются после выполнения `go generate ./...` в проекте.
- **cmd/<имя_модуля>/main.go** — точка входа приложения: запуск сервера, health- и metrics-эндпоинты.
- **pkg/errs/** — типы и утилиты для обработки ошибок (JSON-RPC и HTTP).

После инициализации в корне проекта выполните `go generate ./...`, чтобы сгенерировать транспорт и OpenAPI (например, `api/swagger.json`).
